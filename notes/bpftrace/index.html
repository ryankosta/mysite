KProbe

Functionality

see https://www.kernel.org/doc/html/latest/trace/kprobes.html

  1. KProbe is registered
  2. KProbe makes copy of probed instrn and replaces first byte(s) of probed instrn with breakpoint instrn
  3. when cpu hits said breakpoint instr, trap occurs
    • Executes pre-handler
    • single-steps instrn
    • Executes post-handler
    • continues execution normally

Note: accesses registers so can modify context, including PC

kretprobe

setting up trampoline

  1. Kprobe registered at entry to a fxn
  2. when probe hit, kprobe saves return address of said fxn
  3. kprobe sets up trampoline (typically a no-op) at return addr

At boot time, kprobes register a kprobe @ said trampoline (boot time?, execution time?)

when return instrn executed kprobe executes and

  1. calls kretprobe user-specified return handler
  2. sets saved instrn ptr to actual fxn return addr (so execution continues as normal)
